# VAPT Internship Project - DummyBank

This project demonstrates a complete Vulnerability Assessment and Penetration
Testing (VAPT) workflow against a purpose-built, intentionally vulnerable
sample web application. All data used is dummy/sample data.

## Folder structure

```
VAPT_Project/
├── vulnerable_app/
│   ├── app.py            # Intentionally vulnerable target application (port 5000)
│   └── app_secure.py     # Remediated/fixed version (port 5001)
├── scanner/
│   └── vapt_scanner.py   # Custom Python script that scans for & confirms vulnerabilities
├── results/
│   └── scan_report.txt   # Raw output generated by running the scanner
└── report/
    └── VAPT_Project_Report.docx   # Full project report/documentation
```

## How to run it yourself

1. Start the vulnerable target app:
   ```
   cd vulnerable_app
   python3 app.py
   ```
   It will run at http://127.0.0.1:5000

2. In another terminal, run the scanner:
   ```
   cd scanner
   python3 vapt_scanner.py
   ```
   This prints findings to the console and writes `results/scan_report.txt`.

3. To see the fixed version, stop app.py and run:
   ```
   cd vulnerable_app
   python3 app_secure.py
   ```
   It will run at http://127.0.0.1:5001 — re-running the same attacks against
   it shows they are now blocked.

## Vulnerabilities covered

1. SQL Injection (login form)
2. Sensitive data exposure (exposed database backup)
3. Reflected Cross-Site Scripting (XSS)
4. Broken Access Control / IDOR
5. Weak / default credentials
6. Missing HTTP security headers

Full details, evidence, severity ratings, and remediation steps are in
`report/VAPT_Project_Report.docx`.
